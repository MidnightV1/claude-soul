# claude-soul

[English](README.md) | 中文

让你的 Claude Code 拥有灵魂和记忆。

两个文件。零依赖。Clone 下来重启就生效。

---

## 问题

每次打开 Claude Code，它都是一个陌生人。

不记得你喜欢怎么沟通，不记得上次踩过的坑，不记得你说过"别再给我加没要求的功能了"。你每开一个新会话，就要重新调教一次。

这不是 AI 协作，这是 AI 失忆。

## 方案

**claude-soul** 用两个 Markdown 文件解决这个问题：

| 文件 | 作用 |
|------|------|
| `CLAUDE.md` | **灵魂** — 人格、原则、工作方式、边界。定义 AI "是谁" |
| `COGNITION.md` | **认知** — AI 对你的行为画像。通过观察积累，越用越准。同时也是你审视自己的一面镜子 |

灵魂文件自带久经验证的工程实践默认值，第一天就能用。认知文件从几乎空白开始，随着合作逐步填充——你怎么做决策、什么让你兴奋、什么让你烦、什么时候你想要控制权。

打开认知文件看看 Claude 怎么描述你的——你的决策模式、思维节奏、注意力盲区。有些东西你自己可能从没意识到。这不只是让 AI 更懂你，也是让你更懂自己。

**你不需要填问卷。你只需要正常工作。**

## 30 秒上手

> **已经有 `~/.claude/CLAUDE.md` 了？** 安装前先备份：
> ```bash
> cp ~/.claude/CLAUDE.md ~/.claude/CLAUDE.md.bak
> ```
> Claude 会在首次对话时检测到 `.bak` 文件，问你要不要把旧配置里有用的内容迁移过来。

### 方式一：让 Claude 自己装

打开 Claude Code，说一句话：

```
从 https://github.com/MidnightV1/claude-soul 下载灵魂文件到 ~/.claude/，然后提醒我重启会话。
```

### 方式二：手动安装

```bash
# 备份已有的 CLAUDE.md（如果有的话）
[ -f ~/.claude/CLAUDE.md ] && cp ~/.claude/CLAUDE.md ~/.claude/CLAUDE.md.bak

# 拉取灵魂文件
cd ~/.claude \
  && git init \
  && git remote add origin https://github.com/MidnightV1/claude-soul.git \
  && git fetch origin \
  && git checkout origin/main -- CLAUDE.md COGNITION.md .gitignore
```

**重启 Claude Code 会话**，灵魂即刻生效。

> **关于语言：** 灵魂文件和认知文件默认是英文的（因为英文在模型训练数据中覆盖最广）。首次对话时 Claude 会问你偏好什么语言，之后会用你选择的语言重写这两个文件。

## 它是怎么工作的

### 第一次对话

Claude 读到灵魂文件，发现有几件事还不了解你，会自然地开口问——你的名字、语言偏好、沟通风格、你希望它多大胆。没有冷冰冰的问卷，就像两个人第一次合作时的正常寒暄。

### 之后的每次对话

Claude 启动时加载两个文件。灵魂管行为，认知管上下文。冷启动问题消失了。

### 持续进化

灵魂文件内置了一条元规则：每完成一个任务阶段，反思工作模式。

- 有效的模式 → 沉淀到灵魂或认知文件
- 用覆写替代追加 → 文件始终精炼
- 可选 git 同步 → 灵魂有版本历史

**Claude 会自己进化。你只管用。**

## 云端同步（可选）

如果你想让灵魂跨设备迁移：

1. 创建一个 GitHub 仓库（建议私有，原因见下方提示）
2. 在首次对话时告诉 Claude 启用 git 同步
3. 每次更新后 Claude 自动推送

`.gitignore` 已预配置为白名单模式，只跟踪 `CLAUDE.md`、`COGNITION.md`、`README.md` 和 `.gitignore`。`~/.claude/` 下的凭证、缓存、遥测数据全部排除。

> **隐私提示：** `COGNITION.md` 会逐步积累你的行为画像——决策模式、情绪信号、委托习惯等。如果仓库是公开的，这些数据任何人都能看到。介意的话请使用私有仓库。GitHub 免费账户支持无限私有仓库。

## 文件结构

```
~/.claude/
├── .gitignore       ← 白名单模式，只跟踪灵魂文件
├── CLAUDE.md        ← 灵魂（人格 + 工程实践 + 引导逻辑）
├── COGNITION.md     ← 认知（用户行为画像，持续进化）
└── ...              ← Claude Code 其他文件（被 git 忽略）
```

## 为什么这么设计

**够用就好，别整复杂了。** Claude Code 本身就会读 `~/.claude/CLAUDE.md`，我们只是往里面放了对的内容。不需要额外的框架、不需要启动脚本、不需要学新东西。两个文件解决问题。

**别问，看就行。** 认知文件不靠问卷填充。你正常工作，Claude 在旁边默默观察——你怎么做决策、关注什么忽略什么、什么时候追问什么时候放手。观察到的比问出来的准。

**只留有用的。** 文件更新靠覆写不靠追加。新认知替换旧认知，不是在旁边加一条。用久了文件不会膨胀，只会更精准。

**开箱即用，越用越顺。** 灵魂自带通用工程实践，clone 下来不配置也能用。个性化的部分在日常合作中自然长出来。

## 相关项目

"从对话中积累用户认知"这个思路在 [sovi-demo](https://github.com/MidnightV1/sovi-demo) 的系统提示词中实践了类似的模式：通过静默观察构建用户画像，而非显式收集。claude-soul 将这个模式从特定应用中抽离出来，变成通用的、可移植的 AI 协作基础设施。

[OpenClaw](https://github.com/nicekid1/OpenClaw) 也在探索类似方向——通过文件系统赋予 AI 持久身份。不同的路径，殊途同归。
